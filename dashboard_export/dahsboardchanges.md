# Dashboard Data Corrections

## What Happened

The summary JSONs (`nmf_eng_{5,15,30,30_nm}_summary.json`) were generated by training notebooks that loaded a preprocessed parquet with **3,943 articles**. After those notebooks ran, a `MIN_CONTENT_LENGTH = 200` filter was added to the pipeline's cleaning step (`s02_cleaning.py`). The pipeline correctly filters to **3,939 articles** (4 short FED pages dropped). The analysis-ready CSV, Supabase, and the pipeline all use 3,939. The summary JSONs used by the dashboard were stuck on 3,943 — a different model with different topic assignments.

The discrepancy was not just 4 articles. Because NMF topic boundaries shift when the input changes, topic assignments differed by up to **217 articles per topic** (e.g., teacher_recruitment_and_retention: 248 in old JSON vs 31 in CSV).

## What Was Fixed (in this repo)

1. **Regenerated all 4 summary JSONs** using `scripts/regenerate_summary_jsons.py` — loads the saved NMF models but applies the pipeline's correct preprocessing (3,939 articles). All JSONs now match the CSV and Supabase.

2. **Added `generate_summary_json()` to `s08_save_outputs.py`** — the pipeline now produces summary JSONs automatically on future runs, preventing this from happening again.

3. **Wired into `s10_pipeline.py`** — summary JSON generation is part of the standard pipeline flow.

4. **Updated `SPECIFICATION_SENSITIVITY_REPORT.md`** — all figures verified against the data.
