Production System

Making AI Pipeline Choices Visible

Every AI pipeline makes choices that determine what it finds. These choices are invisible. AtlasED makes them visible and testable. This is a production NLP system analysing education policy documents across England — with every specification choice disclosed and every finding contestable.

Why This Matters This tool will be used to run workshops with policy stakeholders to discuss how AI outputs should be used in high-stakes decision-making. When a model tells you "SEND is the largest education issue," you need to know: is that real, or is that a choice I made when I set k=30?

Why NMF?

Model Choice Rationale
Transparency Over Performance

Non-negative Matrix Factorization (NMF) was chosen deliberately — not because it's the most sophisticated model, but because this project is about specification transparency, and NMF makes every modeling choice explicit and auditable.

The number of topics (k), the vocabulary preprocessing, the document weighting — these aren't hidden inside a neural network. They're parameters you set, and you can see what happens when you change them.

The Core Finding No specification choice preserves more than 18% of topic names. Standard diagnostics cannot detect this. The only way to know is to perturb and compare. That's what the next pages show.
